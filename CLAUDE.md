## Project Overview

**duplicacy-container** is the runtime image and Kubernetes home for the Duplicacy stack around the GeiserX ecosystem.

It exists to package three related concerns cleanly:

1. `drumsergio/duplicacy-container` as the runtime image for Duplicacy jobs
2. `duplicacy-exporter` as a Prometheus metrics source
3. Optional Duplicacy Web UI support when the operator wants a browser-based workflow

## Chart Design Principles

- The backup workload should be Kubernetes-native: schedule the work with `CronJob`, not with an internal daemonized cron process
- The runtime image should stay minimal and script-agnostic
- The exporter should support both `log_tail` and `webhook` modes
- The Web UI must stay optional and generic by default
- `existingSecret` and `existingClaim` support is required wherever practical
- Public docs must clearly explain storage, secret, and licensing assumptions
