# CLAUDE.md — duplicacy-container

## Overview
Container image and Helm chart for running a production-friendly Duplicacy backup stack on Kubernetes. Includes a runtime image for one-shot or cron-style execution, a Helm chart with backup jobs, Prometheus exporter, and optional Web UI.

### Related Projects
- **Sibling**: [GeiserX/duplicacy-cli-cron](https://github.com/GeiserX/duplicacy-cli-cron)
- **Sibling**: [GeiserX/duplicacy-exporter](https://github.com/GeiserX/duplicacy-exporter)

## Tech Stack
- Docker (runtime image)
- Helm 3 (Kubernetes chart)
- Go Template (Helm chart templating)
- Duplicacy CLI (backup engine)
- Prometheus (metrics via duplicacy-exporter)
- Shell (entrypoint scripting)
- GitHub Actions (chart CI and release automation)

## Development
```bash
# Build the runtime image
docker build -t duplicacy-container .

# Install the Helm chart
helm repo add duplicacy https://geiserx.github.io/duplicacy-container
helm install duplicacy duplicacy/duplicacy -f my-values.yaml

# Lint the chart
helm lint charts/duplicacy/

# Template rendering (validation)
helm template charts/duplicacy/
```

## Architecture
- `Dockerfile` + `entrypoint.sh` — Runtime image for Kubernetes-native Duplicacy execution
- `charts/duplicacy/` — Helm chart
  - `Chart.yaml`, `values.yaml`, `values.schema.json`
  - Backup CronJob with PVC support
  - duplicacy-exporter sidecar for Prometheus metrics
  - Optional Web UI pod (disabled by default, port 3875)
  - ServiceMonitor, Ingress, and secrets management
- `docs/` — Documentation and images

## Chart Design Principles
- The backup workload should be Kubernetes-native: schedule with `CronJob`, not with an internal daemonized cron process
- The runtime image should stay minimal and script-agnostic
- The exporter should support both `log_tail` and `webhook` modes
- The Web UI must stay optional and generic by default (no hardcoded image choice)
- `existingSecret` and `existingClaim` support is required wherever practical
- Public docs must clearly explain storage, secret, and licensing assumptions

## Key Rules
- Docker image published to Docker Hub as `drumsergio/duplicacy-container` with semver tags
- Web UI is intentionally disabled by default and kept generic
- Use existing secrets and PVCs; never create credentials in chart values
- Published to ArtifactHub as `duplicacy/duplicacy`
- Designed for one-shot Kubernetes Jobs, not daemonized cron containers
- Validate chart changes with `helm lint` and `helm template`
- Prefer backward-compatible value additions over breaking renames
